Raga Classification App

Audio Processing · Python · Music Analysis

Python · Librosa · Machine Learning · Classical Indian Music

01 / OVERVIEW

PROJECT FOCUS

The Raga Classification App analyzes sequences of musical notes from audio recordings and identifies the raga being performed. It is designed to help students and researchers quickly classify and understand classical Indian music recordings.

The tool leverages audio feature extraction, note quantization, and sequence comparison to match input melodies against a database of known ragas. The app also provides visualizations of note sequences and similarity scores.

02 / TECHNICAL APPROACH

ALGORITHM & IMPLEMENTATION

  • Extracted audio features using Librosa, including pitch, onset, and duration.
  • Transformed audio into discrete note sequences representing the melody.
  • Implemented a Python-based matching algorithm that compares input sequences with stored ragas using pattern alignment and tolerance checks.
  • Developed the `Raga` class to manage note sequences, compare similarity, and handle missing or repeated notes.
  • Visualized results with note timelines, match scores, and a simple GUI interface for quick classification.

Example of note sequence extracted from an audio recording and matched to a known raga.

03 / KEY INSIGHT

CHALLENGES & FINDINGS

Handling ornamentations and microtonal variations in Indian classical music proved challenging. By introducing tolerance in note matching and preprocessing with pitch smoothing, the algorithm maintains high classification accuracy.

Using sequence-based comparisons rather than only pitch histograms allows the app to distinguish ragas with similar note content but different melodic patterns.

05 / LESSONS LEARNED

WHAT I LEARNED BUILDING THIS TOOL

  • Preprocessing audio correctly is critical for accurate note extraction.
  • Sequence-based matching captures melodic patterns better than statistical pitch distributions alone.
  • Python classes like `Raga` make handling complex musical data modular and maintainable.
  • Visualization of intermediate results helps debug and refine classification logic.
  • Handling real-world audio requires careful tolerance and error management for consistent performance.
